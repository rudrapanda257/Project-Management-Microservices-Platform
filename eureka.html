‚öôÔ∏è How Eureka Works Internally

1 Service Registration

When User Service starts, it sends a POST request to Eureka Server:

POST http://localhost:8761/eureka/apps/USER-SERVICE { "instance": { "hostName": "localhost", "app": "USER-SERVICE", "ipAddr": "192.168.1.10", "port": 8081, "status": "UP" } }

‚úÖ Eureka stores this info in its registry

2 Heartbeat / Lease Renewal

Every 30 seconds, User Service sends a heartbeat:

PUT http://localhost:8761/eureka/apps/USER-SERVICE/192.168.1.10:8081 // "Hey Eureka, I'm still alive!"

üíô If no heartbeat received for 90 seconds, Eureka marks service as DOWN

3 Service Discovery

Project Service wants to call User Service:

GET http://localhost:8761/eureka/apps/USER-SERVICE // "Where is USER-SERVICE?" Response: { "instances": [ { "ipAddr": "192.168.1.10", "port": 8081, "status": "UP" } ] }

‚úÖ Project Service now knows where User Service is!

4 Caching

Project Service caches the registry locally and refreshes every 30 seconds

// Local cache in Project Service: { "USER-SERVICE": ["192.168.1.10:8081"], "NOTIFICATION-SERVICE": ["192.168.1.11:8083"] }

‚ö° Fast lookups without constantly querying Eureka!

5 Service Deregistration

When User Service shuts down gracefully:

DELETE http://localhost:8761/eureka/apps/USER-SERVICE/192.168.1.10:8081 // "Goodbye Eureka, I'm shutting down"

‚ùå Eureka removes it from registry immediately
